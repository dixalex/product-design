# Ten Principles for Good Design

- **Author/source:** Dieter Rams, "Ten principles for good design" (c. 1980). Developed during Rams's tenure as chief design officer at Braun. Reproduced at https://www.vitsoe.com/us/about/good-design.
- **Statement:** Ten rules that define the discipline of design rather than the appearance of any object. Coined in the context of consumer electronics and furniture; consequential for software UI because the same failure modes apply.
- **Applies to layer:** Mood (principles #3, #5, #7 — aesthetic discipline, unobtrusiveness, longevity); Polish (principles #8, #10 — thoroughness and restraint as a unity).
- **Why it matters:** Rams names the aesthetic posture that distinguishes refined products — Linear, Stripe Press, Things 3 — from feature-accumulation products. When a Mood gate lands on "calm / refined / restrained," that is Rams's #3 and #5 operating. When a Polish review argues for removing a detail rather than adding one, that is #10. "Less but better" is everywhere; this entry anchors it to the actual principle.
- **How the skill uses it:** Mood layer cites Rams when the user lands a refined or restrained mood — the aesthetic commitment is explicit, not intuitive. Polish layer cites Rams when the user has to choose between adding a detail and removing one; #10 says removal usually wins unless #8 (thoroughness) demands the detail for functional completeness. The two principles are in productive tension at the Polish layer: thorough *and* minimal means every remaining detail must earn its keep.

## Full principle(s)

### 1. Good design is innovative

The one principle the skill rarely fires. Product UI innovation is mostly incremental convention work — nav patterns, form patterns, data-display patterns. #1 is a caution against the opposite failure: reflexive conservatism beyond the point of usefulness. Don't innovate without evidence that convention is failing; don't preserve convention when evidence says otherwise.

### 2. Good design makes a product useful

Utility is prior to appearance. Whether a feature is useful is a product-architecture question, not a visual-design one. But #2 constrains the visual layer: a decorative choice that undermines use — contrast too low for legibility, touch targets too small, animation that delays interaction — fails Rams regardless of aesthetic quality. Visual design that looks refined while making the product harder to use is anti-design.

### 3. Good design is aesthetic

Rams's most important claim: aesthetic quality is not a luxury layered on function but a dimension of function itself. A noisy, incoherent interface imposes cognitive overhead. The Mood layer invokes this when committing to a refined aesthetic — the commitment is not cosmetic. A calm, restrained visual language reduces friction. The aesthetic IS the interface performance.

### 4. Good design makes a product understandable

Typography hierarchy, color semantics, iconographic consistency, and spacing rhythm are legibility tools. This principle underlies the Typography layer's argument for size-and-weight hierarchy (not color hierarchy) and the Color layer's argument for semantic color roles. When a visual choice requires explanation, it has failed #4. Good design is self-evidently legible.

### 5. Good design is unobtrusive

Tools are not jewelry. An unobtrusive interface stays out of the way of the task. This justifies low-decoration defaults, restrained animation, neutral palettes in tool-focused products, and rejecting branding elements that compete with content. The Mood layer fires this when "calm" or "tool-like" is the chosen register.

### 6. Good design is honest

UI doesn't pretend to be what it is not. Fake leather textures, paper-shadow effects, and artificial physicality the software does not have are dishonest. Skeuomorphism is permitted only when the product genuinely replaces a physical artifact — a digital notebook that simulates paper is honest if paper is the product's purpose. Flat, material-neutral UI is the default honest position for software.

### 7. Good design is long-lasting

Written against fashion. The software equivalent: choosing typographic trends — condensed grotesks, oversized hero text, palette fashions — because they look current. Longevity favors proven workhorse typefaces, neutral palettes, and spacing systems based on mathematical ratios rather than trend. The Typography layer's preference for proven faces over fashionable ones is Rams's #7 applied.

### 8. Good design is thorough down to the last detail

"Nothing must be arbitrary or left to chance." This is the Polish layer's primary authority. Hover states, focus rings, disabled state opacity, loading skeletons, transition easing — each could be omitted and each, when omitted, signals inattention. Thoroughness is not elaborateness: every detail the user encounters deserves considered treatment, even if that treatment is a deliberate absence. An invisible focus ring is not minimalism — it is an omission.

### 9. Good design is environmentally friendly

The least-firing principle in software UI. Rams meant material waste and manufacturing impact. Nominal software translation: fewer dependencies, smaller bundles, less JavaScript on low-powered devices. Aligns directionally with "less but better": unnecessary code is waste, the same as unnecessary material.

### 10. Good design is as little design as possible

"Less, but better." The restraint principle; the highest-leverage at the Polish layer. The burden of proof falls on every addition — an animation, a gradient, a border treatment, an icon. The detail must serve clarity or function; if it merely decorates, it should not exist. The tension with #8 is productive: thorough means every necessary detail is considered; minimal means no unnecessary details survive. Together they produce the refined object: complete, not embellished.
